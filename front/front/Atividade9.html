    4- Explique por que o servidor usa o código 201 ao criar um registro:

    código 201 Created avisa explicitamente ao navegador ou aplicativo que a ação não foi apenas processada, mas resultou na criação física e bem-sucedida de um novo recurso no banco de dados.

    5- Escreva o passo a passo completo de um cadastro de usuário, desde o preenchimento do formulário até a resposta do servidor:

  • Preenchimento do Formulário: O usuário acessa a página de cadastro no navegador, digita seus dados e clica no botão "Cadastrar".
  • Validação no Cliente: O navegador verifica rapidamente se o e-mail tem um formato válido e se a senha cumpre os requisitos mínimos, evitando enviar dados obviamente errados para o servidor.
  • Disparo da Requisição HTTP: O navegador empacota os dados digitados em um formato estruturado, monta uma requisição utilizando o método POST e envia esses dados pela internet até o endereço do servidor.
  • Processamento e Validação no Servidor: O servidor recebe a requisição e faz novas checagens de segurança e de regras de negócio. Ele verifica, por exemplo, se aquele e-mail já não está cadastrado no sistema por outra pessoa.
  • Criptografia e Persistência: Caso os dados estejam corretos, o servidor criptografa a senha do usuário e salva o novo registro de usuário dentro do banco de dados.
  • Resposta do Servidor: Após salvar com sucesso, o servidor gera uma resposta HTTP com o status 201 Created, incluindo uma mensagem de confirmação de sucesso.
  • Atualização da Interface: O navegador recebe a resposta do servidor, lê o status de sucesso e atualiza a tela do usuário, geralmente redirecionando-o para a página de login ou para o painel inicial do sistema já autenticado.
